# Feature Plan: State Progression And Personality

## Goal

Make ChuChu feel emotionally legible and memorable through visible state changes and light personality touches.

## User Story

As a player, I want ChuChu to react differently when doing well or poorly so the pet feels alive instead of purely numeric.

## In Scope

- `normal`, `sick`, and `evolved` states
- rules for entering and leaving `sick`
- rules for entering `evolved`
- small easter eggs or personality reactions
- visual distinctions across states

## Out Of Scope

- complex branching evolution trees
- long narrative systems
- dozens of pet expressions

## Experience Intent

The player's care quality should be readable from the pet itself, not only from bars and numbers.

## Definition Of Done

- state changes are rule-based and documented
- each state has a distinct presentation
- at least a few personality moments make ChuChu feel unique
- state recovery is possible and understandable
